# Base Definitions

Common types and structures for civilization game design.

## ID Types

UUID-based unique identifiers for various entities.

## Species

```
SpeciesType {
    speciesTypeID: SpeciesTypeID
    name: string
    description: string
    isEnlightened: bool
    relationships: []SpeciesRelationship
    processes: []ProcessType
}
```

```
SpeciesRelationship {
    targetSpecies: SpeciesTypeID
    favorability: float    // -1.0 to 1.0
    compatibility: float   // 0.0 to 1.0
}
```

```
Species {
    speciesID: SpeciesID
    speciesTypeID: SpeciesTypeID
    quantity: int
}
```

## Processes

The unified system combines production and consumption processes, linking resource transformation with species needs.

```
ProcessType {
    name: string
    description: string
    inputResources: []ResourceChange
    outputResources: []ResourceChange
    processTime: int            // Days to complete
}

ResourceChange {
    resourceType: ResourceTypeID
    buildingID: BuildingID?
    plotID: PlotID?
    size: float
}
```

## Organizations

Organizations represent groups that entities can belong to, including religions, cultures, and potentially clans, families, nations, and alliances in the future.

```
OrganizationType {
    organizationTypeID: OrganizationTypeID
    name: string
    description: string
}

Organization {
    organizationID: OrganizationID
    organizationType: OrganizationTypeID
    name: string
    description: string
    relationships: []OrganizationRelationship
}

OrganizationRelationship {
    targetOrganization: OrganizationID
    favorability: float  // -1.0 (hostile) to 1.0 (friendly)
    attributes: map[string]float  // Additional type-specific attributes (e.g., tolerance, affinity)
}
```

## Religion & Culture

Religions and cultures are specific types of organizations. They maintain backward-compatible interfaces while using the organization system.

```
Religion {
    religionID: ReligionID  // alias for OrganizationID
    name: string
    description: string
    relationships: []ReligionRelationship
}

ReligionRelationship {
    targetReligion: ReligionID
    favorability: float  // -1.0 (hostile) to 1.0 (friendly)
    tolerance: float     // 0.0 to 1.0
}

Culture {
    cultureID: CultureID  // alias for OrganizationID
    name: string
    description: string
    relationships: []CultureRelationship
}

CultureRelationship {
    targetCulture: CultureID
    favorability: float  // -1.0 to 1.0
    affinity: float      // 0.0 to 1.0
}
```
