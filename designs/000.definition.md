# Base Definitions

Common types and structures for civilization game design.

## ID Types

UUID-based unique identifiers for various entities.

## Species

```
SpeciesType {
    speciesTypeID: SpeciesTypeID
    name: string
    description: string
    isEnlightened: bool
    relationships: []SpeciesRelationship
    processes: []ProcessType
}
```

```
SpeciesRelationship {
    targetSpecies: SpeciesTypeID
    favorability: float    // -1.0 to 1.0
    compatibility: float   // 0.0 to 1.0
}
```

```
Species {
    speciesID: SpeciesID
    speciesTypeID: SpeciesTypeID
    quantity: int
}
```

## Processes

The unified system combines production and consumption processes, linking resource transformation with species needs.

```
ProcessType {
    name: string
    description: string
    inputResources: []ResourceChange
    outputResources: []ResourceChange
    processTime: int            // Days to complete
}

ResourceChange {
    resourceType: ResourceTypeID
    unitID: UnitID?
    plotID: PlotID?
    size: float
}
```

## Resource

```
ResourceType {
    resourceTypeID: ResourceTypeID
    name: string
    description: string
    resourceEfficiencys: []ResourceEfficiency
}
```

```
ResourceEfficiencyType {
    resourceEfficiencyTypeID: ResourceEfficiencyTypeID
    name: string // e.g., "Food", "Tool", "Construction", "Energy", "Luxury"
    description: string
}
```

```
ResourceEfficiency {
    resourceEfficiencyTypeID: ResourceEfficiencyTypeID
    rate: float
}
```

## Unit

Units are generalized entities that can represent various types of active entities including buildings, caravans, missions, armies, etc.

```
UnitType {
    unitTypeID: UnitTypeID
    name: string
    description: string
    processes: []ProcessType
    storages: []StorageType
    workerRequirement: []WorkerRequirement
}
```

```
WorkerRequirement {
    speciesTypeID: SpeciesTypeID
    skills: []Skill
    size: int
}
```

```
Unit {
    unitID: UnitID
    unitTypeID: UnitTypeID
    ownerClusterID: ClusterID?
    workers: []speciesID
}
```

## PlotAttributes

```
PlotAttributes {
    ownerClusterID: ClusterID?
    terrainType: TerrainTypeID
    neighborPlots: []PlotID
    storages: []StorageType
    units: []Unit
    species: []Species
    populations: []Population
}
```

```
TerrainType {
    terrainTypeID: TerrainTypeID
    name: string
    description: string
    processes: []ProcessType
}
```

## StorageType

```
StorageType {
    size: float
    capacity: float
    resourceType: ResourceTypeID
}
```

## Population

```
Population {
    speciesID: SpeciesID
    clusterIDs: []ClusterID
    skills: []Skill
    size: int
}
```

## Skill

```
SkillType {
    skillTypeID: SkillTypeID
    name: string
    description: string
}
```

```
Skill {
    skillTypeID: SkillTypeID
    experience: float
}
```

## Clusters

Clusters represent groups that entities can belong to, including religions, cultures, and potentially clans, families, nations, and alliances in the future.

```
ClusterType {
    clusterTypeID: ClusterTypeID
    name: string
    description: string
}

Cluster {
    clusterID: ClusterID
    clusterType: ClusterTypeID
    name: string
    description: string
    relationships: []ClusterRelationship
}

ClusterRelationship {
    targetCluster: ClusterID
    favorability: float  // -1.0 (hostile) to 1.0 (friendly)
}
```
