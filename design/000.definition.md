# Base Definitions

Common types and structures for civilization game design.

## ID Types (UUID-based)

```
PlayerID, PlotID, BuildingID, CultureID, ReligionID
ResourceTypeID, SpeciesTypeID, TerrainTypeID, BuildingTypeID
```

Note: Population uses SpeciesTypeID (population is merely a civilized species).

## Coordinate & Calendar Systems

```
Point { x: float, y: float }  // Abstract world units

// Earth calendar
daysPerYear: int = 365
monthsPerYear: int = 12
daysPerMonth: int = 30  // Approximate
```

## Religion & Culture Systems

```
Religion {
    id: ReligionID
    name: string
    relationships: []ReligionRelationship
}

ReligionRelationship {
    targetReligion: ReligionID
    favorability: float  // -1.0 (hostile) to 1.0 (friendly)
    tolerance: float     // 0.0 to 1.0
}

Culture {
    id: CultureID
    name: string
    relationships: []CultureRelationship
}

CultureRelationship {
    targetCulture: CultureID
    favorability: float  // -1.0 to 1.0
    affinity: float      // 0.0 to 1.0
}
```

## Consumption System

Species consume resources for various needs. Consumption affects mood and reproduction.

```
NeedType: FOOD, TOOLS, SHELTER, LUXURY, MEDICINE, EDUCATION

ConsumptionNeed {
    needType: NeedType
    requiredAmount: float           // Per capita per day
    satisfactionThreshold: float    // 0.0 to 1.0
    impactOnMood: float            // -1.0 to 1.0
    impactOnReproduction: float
}

ResourceAttribute {
    needType: NeedType
    effectivenessValue: float  // How well it satisfies need (0.0 to 1.0)
}
```

## Species System

```
Species {
    id: SpeciesTypeID
    name: string
    isEnlightened: bool
    relationships: []SpeciesRelationship
    needs: []ConsumptionNeed
}

SpeciesRelationship {
    targetSpecies: SpeciesTypeID
    favorability: float    // -1.0 to 1.0
    compatibility: float   // 0.0 to 1.0
}
```

## Resource System

Produced by plots/buildings, consumed by species.

```
ResourceType {
    id: ResourceTypeID
    name: string
    category: ResourceCategory  // RAW_MATERIAL, PROCESSED, ENERGY, ABSTRACT
    attributes: []ResourceAttribute
    buildingMaterial: bool
    energySource: bool
    energyValue: float
    buildingEfficiency: float  // 0.0 to 1.0
    conversions: []ResourceConversion
}

ResourceConversion {
    outputResource: ResourceTypeID
    conversionRate: float
    requiredBuilding: BuildingTypeID?
}
```

## Terrain System

```
TerrainType {
    id: TerrainTypeID
    name: string
    movementCost: float
    defensiveBonus: float
    fertility: float  // 0.0 to 1.0
    resourceYield: []ResourceYield
    buildable: bool
    navigable: bool
}

ResourceYield {
    resourceType: ResourceTypeID
    baseYield: float
    extractionDifficulty: float  // 0.0 to 1.0
}
```

## Building System

```
BuildingType {
    id: BuildingTypeID
    name: string
    category: BuildingCategory  // ECONOMIC, MILITARY, CULTURAL, RELIGIOUS, INFRASTRUCTURE
    constructionCost: []ResourceCost
    maintenanceCost: []ResourceCost
    requiredTerrain: []TerrainTypeID
    workersRequired: int
    effects: []BuildingEffect
}

ResourceCost {
    resourceType: ResourceTypeID
    amount: float
}

BuildingEffect {
    type: EffectType  // RESOURCE_PRODUCTION, RESOURCE_CAPACITY, DEFENSE_BONUS, etc.
    targetResource: ResourceTypeID?
    magnitude: float
}
```

## Notes

- IDs: UUID format
- Arrays: [] prefix, Optional: ? suffix
- Ranges: favorability/impact (-1.0 to 1.0), percentages (0.0 to 1.0)
- Resources can satisfy multiple needs with varying effectiveness
